# VCAM · 安卓虚拟摄像头（合成模式增强）

基于 Xposed 的虚拟摄像头模块。除了把相机预览/拍照替换成自定义视频、图片，本分支的重点是 **合成模式**：以**真实相机画面为背景**，把你的图片/视频当作「挂件」叠加上去，可拖拽定位、缩放、任意角度旋转，合成后的画面再推送给目标 App。

> 本项目由 [@w2016561536](https://github.com/w2016561536/android_virtual_cam) 的原版及 [Cross2pro](https://github.com/Cross2pro/android_VCAM-Revise) 的 Revise 版二次开发而来，已与原项目差异较大。

## 🌟 特性

**合成模式（本分支核心）**

- 🎬 **真实相机 + 挂件合成**：真实相机当背景，图片/视频当挂件叠加，GL 实时合成后推给 App
- 🖐️ **可视化拖拽**：9:16 画布上拖动挂件定位，滑块调尺寸
- 🔄 **任意角度不失真**：挂件可 0°/90°/180°/270° 旋转，框的宽高比随旋转自动对调，画面不压扁
- 🎯 **按 App 预设相机方向**：选目标 App 后自动校正相机方向（抖音 0°、TikTok 90°），并保留手动「相机旋转」兜底

**替换能力**

- 📹 **视频替换**：将相机预览替换为自定义 MP4
- 🖼️ **照片替换**：将拍照结果替换为自定义图片
- 📡 **直播源替换**：填 HTTP-FLV 直播地址，目标 App 一开相机即拉流（含音频）
- 🔄 **本地视频实时旋转**：旋转在 Hook 内实时完成，长视频不再分钟级预转码，秒生效
- 🔊 **音频播放**：可选播放视频/挂件原声

**其它**

- 🎨 底部 Tab 分「合成 / 设置」两页，目标 App 选择精简为「已安装且申请了相机权限」的应用
- 📁 强制私有目录默认开启，各 App 配置互不干扰
- ⚡ 配置实时生效，无需重启模块

## 📱 支持平台

- Android 5.0+ (API 21+)
- 需要设备已 **root** 并安装 **[LSPosed](https://github.com/JingMatrix/LSPosed)** 框架
- 合成模式 /「应用合成」通过 su 写入目标 App 私有目录，同样依赖 root

## 📦 安装

前置：设备已 root，并已安装 [LSPosed](https://github.com/JingMatrix/LSPosed)（Magisk / KernelSU 模块）。

1. 安装本模块 APK
2. 打开 **LSPosed 管理器 → 模块**，启用 **VCAM**
3. 在模块的**作用域**里勾选要替换相机的目标 App（**无需勾选系统框架**）
4. 强制停止并重新打开目标 App，使模块生效

## 📖 使用

### 合成模式（推荐）

打开 VCAM，底部切到「合成」页，四步完成：

1. **选目标 App** —— 从列表选择，自动按 App 校正相机方向（抖音 0°、TikTok 90°）
2. **选挂件** —— 选一张图片或一段视频作为叠加挂件
3. **调位置和大小** —— 在 9:16 画布上拖动挂件、滑块调尺寸；需要时点「挂件旋转」「相机旋转」
4. **应用合成到目标 App** —— 强制停止目标 App 后重开，即可看到「真实相机 + 挂件」的合成画面

> 💡 抖音「拍摄」与「开直播」入口相机方向不同，切过去后点几下「相机旋转」再重新「应用合成」即可。

「应用合成」会用 root 把挂件与配置写入目标 App 的私有目录 `Android/data/[包名]/files/Camera1/`。

### 传统视频 / 照片替换

若只想整屏替换而非合成：

1. 在目标 App 打开相机预览，留意 Toast 提示的分辨率（宽×高）
2. 制作同分辨率的替换视频，命名为 `virtual.mp4`
3. 放入工作目录（见下）
4. 拍照替换：准备同分辨率图片命名 `1000.bmp` 放入同目录（拍照时有 Toast 提示才需要）

**工作目录**

- 私有目录（默认，推荐）：`/[内部存储]/Android/data/[包名]/files/Camera1/`
- 共享目录：`/[内部存储]/DCIM/Camera1/`

### 直播源替换

填入 `http(s)://` 开头的 HTTP-FLV 地址应用后，目标 App 一开相机即拉流。仍需一个本地视频作占位以激活 Hook。

### 设置

「设置」页两个开关：

- **播放视频原声**：播放替换视频/挂件的音频
- **临时停用模块**：临时停用替换（开=停用）

> 💡 强制私有目录已默认开启，无需手动开关。两开关也可在 `/[内部存储]/DCIM/Camera1/` 下手动建标记文件：`no-silent.jpg`（原声）、`disable.jpg`（停用）。

## ❓ 常见问题

- **画面方向不对？** 合成模式点「相机旋转」/「挂件旋转」调整；传统替换需用剪辑软件调整原视频方向与分辨率。前置摄像头通常需水平翻转并旋转。
- **黑屏 / 相机启动失败？** 部分 App（尤其系统相机）无法替换；或视频路径不对（注意别建成两级 `Camera1/Camera1/`）。
- **画面花屏 / 变形？** 视频分辨率不匹配，按 Toast 提示的分辨率重新制作。
- **抖音替换无效？** 抖音采集 Surface 走 preview 通道，本分支已适配；若仍无效确认已强制停止 App 后重开。

## 🔧 构建

- **当前版本**：v4.4 (versionCode 28)
- **编译 SDK**：Android 12 (API 31)，最低 API 21，目标 API 26

```bash
# 必须用 JDK 17（JDK 21 会导致 Gradle 7.5 报 Unsupported class file major version 65）
export JAVA_HOME=/path/to/jdk-17

./gradlew assembleDebug     # 调试版
./gradlew assembleRelease   # 发布版（需签名配置）
```

> ⚠️ 构建配置里已通过剥离调试信息（`-g:none`）绕开 D8 3.1.51 在部分匿名类上的 dexing NPE，请勿移除。
